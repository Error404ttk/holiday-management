# คู่มือผู้ดูแลระบบ

คู่มือนี้สำหรับ admin และ super_admin ที่มีสิทธิ์จัดการวันหยุด

## ก่อนแก้ไขข้อมูลจริง

1. ตรวจว่า backup ตาราง `hosxp.holiday` แล้ว
2. ตรวจว่า backup ฐาน app ที่เก็บ `users` และ `audit_logs` แล้ว
3. ตรวจว่าเลือกปีและช่วงวันที่ถูกต้อง
4. ตรวจประกาศวันหยุดหรือเอกสารอ้างอิงของหน่วยงาน
5. หลีกเลี่ยงการแก้ไขช่วงที่มีผู้ใช้งานระบบ HOSxP จำนวนมาก

## เพิ่มวันหยุด

1. ไปที่เมนู “รายการวันหยุด”
2. กด “เพิ่มวันหยุด”
3. กรอกวันที่วันหยุด
4. กรอกชื่อวันหยุด
5. เลือกประเภทวันหยุด
6. เลือกสถานะใช้งาน
7. กรอกหมายเหตุถ้ามี
8. กด “บันทึก”
9. ตรวจข้อมูลใน confirm dialog
10. กดยืนยันบันทึก

ถ้าวันที่ซ้ำ ระบบจะแจ้งเตือนหรือ block ตาม validation ฝั่ง backend

## แก้ไขวันหยุด

1. ไปที่เมนู “รายการวันหยุด”
2. กดไอคอนดินสอในรายการที่ต้องการแก้
3. แก้ข้อมูลที่ต้องการ
4. กรอก “เหตุผลในการแก้ไข”
5. กด “บันทึก”
6. ตรวจข้อมูลใน confirm dialog
7. กดยืนยันบันทึก

ระบบจะบันทึก `old_value` และ `new_value` ลง audit log

## ปิดใช้งานวันหยุด

ฟีเจอร์นี้ใช้ได้เฉพาะเมื่อตาราง `holiday` มี field active/status และตั้งค่า `HOLIDAY_ACTIVE_COLUMN` แล้ว

1. ไปที่เมนู “รายการวันหยุด”
2. กดไอคอนปิดใช้งาน
3. กรอกเหตุผลในการปิดใช้งาน
4. กดยืนยัน

ระบบไม่ลบข้อมูลจริง และไม่มี DELETE endpoint

## ดู Audit Log

1. เลือกเมนู “ประวัติการแก้ไข”
2. ดูรายการตามวันเวลา ผู้ใช้งาน action ตาราง และ IP Address
3. ใช้ตัวกรองผู้ใช้งาน, action, วันที่เริ่มต้น และวันที่สิ้นสุด
4. กดไอคอนรูปตาเพื่อดูรายละเอียด

## อ่าน old_value / new_value

- `old_value`: ข้อมูลก่อนแก้ไข
- `new_value`: ข้อมูลหลังแก้ไข
- `reason`: เหตุผลที่ผู้ใช้งานกรอกตอนแก้ไขหรือปิดใช้งาน
- `user_agent`: browser หรือ client ที่เรียก API

## ข้อควรระวังเรื่อง HOSxP

- ระบบนี้เขียนเฉพาะตาราง `hosxp.holiday`
- ห้ามเพิ่มสิทธิ์ DELETE ให้ DB user ของระบบ
- ห้ามแก้ schema HOSxP จากระบบนี้
- ก่อน UAT หรือใช้งานจริงต้อง backup เสมอ
- ถ้า audit log เขียนไม่สำเร็จ ระบบจะ rollback การเขียน holiday
- ถ้าระบบแจ้งว่า primary key ไม่ชัดเจน ให้หยุดใช้งาน write path และตรวจ `SHOW COLUMNS FROM holiday`
